Cookie and Token Manipulation

The definition of Cookie and Token Manipulation

According to Radware, an American cybersecurity company, Cookie Manipulation (Also called Cookie Poisoning) is the act of manipulating or forging a cookie (a small piece of data created and stored in a user's browser that keeps track of important information regarding their session information for a particular site) for the purpose of bypassing security measures or sending false information to a server.

Usage in the cybersecurity world

Cookie-Based authentication uses HTTP Cookies to authenticate client requests using stateless HTTP protocol.

  1. When a client sends a request to login, it goes to the backend server, the server verifies their credentials, and, if successful, a Set-Cookie is created.
  2. This cookie is saved locally, and the user needs this cookie to remain logged in.
  3. The cookie expires when the user logs out.
